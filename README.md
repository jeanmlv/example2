# example2

My workflow consists of four main steps:

1. Verify data availability

I first check which ADaM datasets are available in Domino for the study.

2. Generate the ARD

Using the available ADaM datasets, I generate the ARD following the required structure.

3. Validate and publish

After validating the generated ARD, I publish the final version to the Thea View repository.

4. Assess missing variables

Finally, I compare the available variables against the predefined list of variables of interest. This allows me to identify which variables are available in the current ADaM package and which ones are missing, so they can be requested if needed.

Se alguém perguntar especificamente "How do you identify the missing variables?", você pode responder:

The missing variables are identified after the ARD is generated. I compare the predefined list of variables of interest with the variables actually available in the ADaM datasets used to build the ARD. Any required variable that is not available is documented as a missing variable for that study.

The process is essentially sequential. First, I check which ADaM datasets are available in Domino for the study. Based on those datasets, I generate and validate the ARD, and then publish the final version to the Thea View repository. Once the ARD is available, I compare the variables of interest with the variables actually present in the ADaM datasets.
